# Final Report

At the end of a task, produce a precise report.

## Required sections

1. Summary
2. Files changed
3. Commands run
4. Tests run
5. Results
6. Risks remaining
7. Manual checks required
8. Deployment instructions if relevant
9. Rollback instructions if relevant
10. Exact next command for the user

## Rules

* Be factual.
* Do not overclaim.
* Do not hide failed tests.
* Do not say production-ready unless all production criteria are verified.
* If no tests were run, say so explicitly.
